## Development Guidelines

### Adding New Icons

1. **Design Consistency**: Follow existing design language
2. **Theme Alignment**: Assign appropriate theme based on use case (banking/crypto/defi/investing/neutral)
3. **Accessibility**: Use `currentColor` and proper stroke properties
4. **Testing**: Test across different sizes and themes

### Icon Design Standards

- Use 24x24 viewBox as default
- Stroke width of 2 for consistency
- Use `stroke="currentColor"` for theming
- Include `stroke-linecap="round"` for better appearance
- Avoid complex fills unless necessary

### Performance Considerations

- Icons bundled as SVG strings
- Tree shaking eliminates unused icons
- Gzip compression reduces payload size
- Consider icon splitting for large applications
